# E-Commerce Analytics & Customer Segmentation Dashboard

## About the Project

This project is analysis of e-commerce sales and customer data.

The main idea behind the project was to take raw transaction data and understand what is actually happening in the business. I wanted to answer questions such as:

- How much revenue is the business generating?
- How many orders and customers are there?
- How are sales changing over time?
- Which products generate the most revenue?
- Which countries contribute the most to sales?
- Who are the most valuable customers?
- Can customers be grouped based on their purchasing behaviour?

To answer these questions, I used **Python for data cleaning and customer segmentation, MySQL and SQL for business analysis, and Power BI for building an interactive dashboard**.

---

## Dataset

I used the Online Retail dataset, which contains transaction data from an online retail business.

The original dataset had more than 500,000 rows and included information such as:

- Invoice number
- Product code
- Product description
- Quantity purchased
- Invoice date
- Unit price
- Customer ID
- Customer country

The raw data could not be directly used for analysis because it contained missing customer IDs, duplicate rows, cancelled orders, and invalid quantities.

---

## What I Did

The project was divided into four main parts:

1. Data cleaning and preparation using Python
2. Sales and business analysis using SQL
3. Customer segmentation using RFM analysis
4. Dashboard development using Power BI

---

## 1. Data Cleaning Using Python

I first loaded the raw Excel dataset using Pandas and checked the number of rows, column data types, and missing values.

During data cleaning, I:

- Removed duplicate rows
- Removed records with missing Customer IDs
- Removed cancelled invoices
- Removed transactions with zero or negative quantities
- Removed transactions with invalid unit prices

I then created a new `Revenue` column.

The revenue for each transaction was calculated as:

`Revenue = Quantity × Unit Price`

I also extracted useful information from the invoice date, including:

- Year
- Month
- Month name
- Day
- Hour

These columns were created so that sales could later be analyzed by month, year, and time of day.

After cleaning, the dataset contained **392,692 valid transaction records**.

The cleaned data had:

- 18,532 unique orders
- 4,338 unique customers
- Approximately 8.89 million in total revenue

The Python data cleaning code is available in `data_cleaning.py`.

---

## 2. Sales Analysis Using SQL

After cleaning the data, I imported it into MySQL and used SQL to analyze business performance.

I first calculated four main business KPIs:

- Total Revenue
- Total Orders
- Total Customers
- Average Order Value

Since one invoice can contain multiple products, I counted distinct invoice numbers to calculate the actual number of orders.

Average Order Value was calculated as:

`Average Order Value = Total Revenue / Total Orders`

I also wrote SQL queries to analyze:

- Monthly revenue trends
- Top 10 products by revenue
- Revenue by country
- Top customers based on total spending
- Number of orders by hour
- Customer purchase behaviour

The goal of the SQL analysis was not just to query the data, but to answer common business questions using transaction data.

All SQL queries used in the project are available in `ecommerce_analysis.sql`.

## 3. Customer Segmentation Using RFM Analysis

The next part of the project was understanding customer purchasing behaviour.

Instead of treating every customer in the same way, I used **RFM analysis** to group customers based on their transaction history.

RFM stands for:

### Recency

Recency measures how recently a customer made a purchase.

A customer who purchased recently is generally more active than a customer who has not purchased for a long time.

### Frequency

Frequency measures how often a customer places an order.

I calculated this by counting the number of unique invoices for each customer.

### Monetary

Monetary value measures how much money a customer has spent.

I calculated the total revenue generated by each customer.

---

### RFM Scoring

For each customer, I calculated Recency, Frequency, and Monetary values.

Customers were then given scores from 1 to 4 for each RFM metric.

The scores were combined to create an overall RFM score.

Based on the final score, customers were divided into four groups:

- **High Value** — Customers with high purchasing activity and spending
- **Loyal** — Customers who purchase regularly
- **Regular** — Customers with moderate purchasing behaviour
- **At Risk** — Customers with lower engagement or purchasing activity

The purpose of this segmentation was to identify different types of customers and understand which groups are more valuable to the business.

For example, high-value customers can be targeted with exclusive offers, while at-risk customers can be targeted with re-engagement campaigns.

The customer segmentation code is available in `customer_segmentation.py`.

The final segmentation results are stored in `customer_segments.csv`.

## 4. Power BI Dashboard

Finally, I built an interactive Power BI dashboard to present the analysis.

I imported the cleaned transaction data and customer segmentation data into Power BI.

The two tables were connected using `CustomerID`.

I created DAX measures for:

- Total Revenue
- Total Orders
- Total Customers
- Average Order Value

These measures automatically update when dashboard filters are applied.

---

## Dashboard Visuals

The dashboard contains four KPI cards:

- Total Revenue

- Total Customers
- Average Order Value

I also created the following visualizations:

### Monthly Revenue Trend

A line chart showing how revenue changes across different months.

A bar chart comparing revenue generated across the top-performing countries.

## Interactive Filters

- Year
- Country

For example, selecting the **High Value** customer segment updates the dashboard and shows the revenue, orders, product performance, and country performance specifically for high-value customers.

This makes it easier to analyze different customer groups and business areas.
